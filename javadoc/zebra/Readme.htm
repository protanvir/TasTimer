Description

RFID EMDK for Java provides a set of tools necessary to develop Java applications for RFID Reader devices from Zebra. These tools include class libraries, sample applications, and associated documentation. EMDK for Java allows Java developers to programmatically access the RFID reader functionality. The application can be developed to run on the host PC.

RFID EMDK Java should only be used for devices FX Series Readers (FX7500, FX9600)

For more information on Zebra enterprise mobility devices, please visit:

            http://www.zebra.com

Sample Applications

J_RFIDHostSample1

The host based sample application runs on the Desk top PC to connect Zebra RFID Reader (MC series/FX/XR) based on the IP/Host Name. This app demonstrates the complete functionality using Java RFID API3 class library (Symbol.RFID.API3.jar)

Access the Reader Connect dialog from Reader->Connection Menu

ï¿½         To connect FX/XR Reader, provide the respective IP address and port number (default: 5084).

ï¿½         To connect MC Series Reader, provide the IP adress as 169.254.2.1 and port number (default: 5084).

 

This app helps to demonstrate the following functions of RFID Reader:

  • Secure Connection
  • Gets the Reader capabilities
  • Reader configuration
  • Operation (Inventory & Access Operation Read, write, lock, kill, block erase and block write)
  • Filters (Pre-Filter, Post Filter and Access Filter)
  • Triggers (Start and Stop Condition
  • Reader management functions

Known Issues

  1. Accept connection not yet implemented (In this scenario, Reader will be client and application will act as server. Reader will attempt to make a connection to the application)
    2.
  2. Negative parameter validation is not done in  the API. Wherever applicable, the user is expected to provide non-negative values as parameter.
  3. During un-installation of Zebra RFID FXSeries Host Java SDK v1.4, the RFIDAPI32PC.dll in the windows system folder will not be deleted. This is to work with other EMDK for C or .NET product
